## 🚀 Terraform Important Commands Cheat Sheet

| 🔢 No  | ⚡ Terraform Command                     | 🧠 Detailed Explanation                                                                                                                          | 🔍 Common Use Case                        |
| ------ | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------- |
| 1️⃣    | 🚀 `terraform init`                     | 📦 Initializes Terraform working directory by downloading provider plugins, modules, and backend configurations. Must run before other commands. | 🏁 Start new Terraform project            |
| 2️⃣    | 📋 `terraform plan`                     | 🔍 Creates execution plan showing what Terraform will add, change, or destroy before applying infrastructure changes.                            | 👀 Preview infrastructure changes         |
| 3️⃣    | ✅ `terraform apply`                     | ⚙️ Applies infrastructure changes required to match desired Terraform configuration state.                                                       | 🏗️ Create/update infrastructure          |
| 4️⃣    | 💥 `terraform destroy`                  | 🗑️ Removes all infrastructure resources managed by Terraform configuration.                                                                     | ❌ Delete environments/resources           |
| 5️⃣    | ✔️ `terraform validate`                 | 🧪 Checks Terraform configuration syntax and validates whether configuration is internally consistent.                                           | 🔎 Verify configuration correctness       |
| 6️⃣    | 🎨 `terraform fmt`                      | ✨ Automatically formats Terraform configuration files into canonical style for readability and consistency.                                      | 🧹 Clean and standardize code             |
| 7️⃣    | 📖 `terraform show`                     | 📄 Displays current Terraform state or contents of saved execution plan in human-readable format.                                                | 🔍 Inspect state or plan                  |
| 8️⃣    | 🔄 `terraform refresh`                  | 🌐 Updates Terraform state file with latest real-world infrastructure changes from providers.                                                    | 🔃 Sync state with infrastructure         |
| 9️⃣    | 📊 `terraform graph`                    | 🕸️ Generates dependency graph of Terraform resources in DOT format for visualization.                                                           | 📈 Visualize resource relationships       |
| 🔟     | 📤 `terraform output`                   | 🧾 Displays output variables defined in Terraform configuration after deployment.                                                                | 🔑 Retrieve infrastructure values         |
| 1️⃣1️⃣ | 🔌 `terraform providers`                | 📚 Lists providers required by current Terraform configuration and modules.                                                                      | 🛠️ Check provider dependencies           |
| 1️⃣2️⃣ | 📃 `terraform state list`               | 🗂️ Shows all resources currently tracked in Terraform state file.                                                                               | 👁️ View managed resources                |
| 1️⃣3️⃣ | 🔍 `terraform state show <address>`     | 📑 Displays detailed attributes and metadata of specific resource in Terraform state.                                                            | 🧠 Inspect resource details               |
| 1️⃣4️⃣ | 🚚 `terraform state mv <src> <dest>`    | 🔀 Moves or renames resource entries within Terraform state without recreating infrastructure.                                                   | 🏷️ Refactor resource names               |
| 1️⃣5️⃣ | ❎ `terraform state rm <address>`        | 🧹 Removes resource from Terraform state without deleting actual infrastructure resource.                                                        | 🚫 Stop managing resource                 |
| 1️⃣6️⃣ | 📥 `terraform import <address> <id>`    | 🌍 Imports existing infrastructure resources into Terraform state management.                                                                    | 🔄 Bring manual resources under Terraform |
| 1️⃣7️⃣ | ⚠️ `terraform taint <address>`          | ♻️ Marks resource as tainted so Terraform recreates it during next apply operation.                                                              | 🔧 Force resource recreation              |
| 1️⃣8️⃣ | 🛡️ `terraform untaint <address>`       | ✅ Removes taint status from resource to prevent forced recreation.                                                                               | 🚫 Cancel resource recreation             |
| 1️⃣9️⃣ | 📂 `terraform workspace list`           | 🌐 Lists all Terraform workspaces available in current configuration.                                                                            | 🏢 View environments/workspaces           |
| 2️⃣0️⃣ | 🆕 `terraform workspace new <name>`     | 🏗️ Creates new Terraform workspace for isolated state management.                                                                               | 🌍 Create DEV/UAT/PROD environments       |
| 2️⃣1️⃣ | 🔄 `terraform workspace select <name>`  | 🎯 Switches active Terraform workspace.                                                                                                          | 🔀 Change environments                    |
| 2️⃣2️⃣ | 🗑️ `terraform workspace delete <name>` | ❌ Deletes Terraform workspace and its associated state.                                                                                          | 🧹 Remove unused environment              |
| 2️⃣3️⃣ | 🔐 `terraform login`                    | ☁️ Authenticates user with Terraform Cloud or Terraform Enterprise.                                                                              | 🌩️ Access Terraform Cloud                |
| 2️⃣4️⃣ | 🚪 `terraform logout`                   | 🔓 Removes authentication credentials for Terraform Cloud/Enterprise.                                                                            | 🔒 Secure logout                          |
| 2️⃣5️⃣ | 🏢 `terraform cloud organization list`  | 📋 Lists Terraform Cloud organizations accessible to authenticated user.                                                                         | 👥 Manage Terraform Cloud orgs            |
| 2️⃣6️⃣ | ☁️ `terraform cloud workspace list`     | 📂 Lists Terraform Cloud workspaces in organization.                                                                                             | 🌐 View cloud workspaces                  |
| 2️⃣7️⃣ | 🧾 `terraform version`                  | 🔢 Displays installed Terraform CLI version and provider versions.                                                                               | 🛠️ Verify Terraform version              |
| 2️⃣8️⃣ | 🔒 `terraform providers lock`           | 📌 Generates dependency lock file (`.terraform.lock.hcl`) for consistent provider versions.                                                      | 🧷 Provider version control               |
| 2️⃣9️⃣ | 🔓 `terraform force-unlock <lock-id>`   | 🚨 Manually unlocks Terraform state file when state lock becomes stuck.                                                                          | 🆘 Recover locked state                   |
| 3️⃣0️⃣ | 💻 `terraform console`                  | 🧠 Opens interactive Terraform console for testing expressions and variables.                                                                    | 🧪 Evaluate Terraform expressions         |
